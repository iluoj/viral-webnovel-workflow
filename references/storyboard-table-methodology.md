# 分镜表构建方法论

> ⚠️ 蒸馏自 Toonflow `production_execution_storyboard_table.md`（HBAI-Ltd/Toonflow-app，Apache-2.0），为 viral-webnovel 适配。
> 用途：drama-adapt（△剧本）→ prompt-gen（麻薯提示词）之间的新增中间步骤。把△剧本结构化拆分为分镜表，再转提示词。

---

## 分镜表在流水线中的位置

```
△剧本 → [导演规划 → 分镜表] → 麻薯分镜提示词
               ↑ 本次新增
```

分镜表 = 结构化中间产物。每一行是 ≈15 秒的一个片段，包含：画面描述/场景/景别/角色动作/情绪/光影/台词/音效。

---

## 核心铁律（优先级从高到低）

1. **台词零删改**：剧本所有引号内台词、VO、旁白必须 100% 逐字搬运
2. **出场人物完整**：剧本列出的人物不能遗漏
3. **只描述动作状态**：画面描述只写动作/姿态/表情/状态变化
4. **长台词/长VO强制拆镜**：超过 20 字必须拆多个连续镜头，每镜换视角/景别

---

## 分镜表列结构

| 列 | 说明 | 来源 |
|:--|:-----|:-----|
| **序号** | 当前场内的镜号 | 自动编号 |
| **画面描述** | 谁做了什么动作/姿态/表情/状态变化。**不写服装/发型/长相**（这些交给资产） | 从△剧本的动作描述 + 场景描述转化 |
| **场景** | 场景名称（与场景资产表对齐） | 从△剧本的场景头提取 |
| **景别** | 大远景 / 远景 / 中景 / 近景 / 半身 / 特写 / 大特写 / 过肩镜 | 自行设计 |
| **运镜** | 缓推/拉远/平移/固定/手持/跟随等（仅作分镜信息，不进入提示词） | 自行设计 |
| **角色动作** | 主体动作 + 朝向（`｜朝向：3/4正面朝右`） | 从△剧本的动作描述提取 |
| **情绪** | 情绪基调关键词 | 从△剧本的情绪/语气提取 |
| **光影氛围** | 光源方向/色调倾向/明暗关系 | 从场景描述 + 时间信息推导 |
| **音效** | 环境音 + 音效（**不写 BGM/配乐**） | 从△剧本的音效标记提取 |
| **台词** | 100% 原文搬运 | 从△剧本对白复制 |
| **时长** | 该片段时长（秒），≤15 秒/片段 | 按 4字/秒 计算台词时长 |

---

## 分镜设计规范

### 1. 每片段 ≤15 秒

- 单片段台词字数超载时，拆分为多个片段
- 拆分时按语义停顿点切，不平均切分
- 台词可以第一镜完整写出，跨镜可切到他人反应镜

### 2. 长台词/长VO强制拆镜

- 长台词或长VO超过 20 字必须拆多个连续镜头
- 每镜换视角/景别，按语义停顿点切
- 若语义不可切必须单镜呈现，用表情微妙变化/肢体持续演进/运镜缓慢推动填满时间
- 禁止单镜固定不动

### 3. 出场人物不能消失

- 先看清剧本的出场人物列表，分镜里一个都不能漏
- 剧本没写「XX 离开」，XX 就还在场，必须有视觉痕迹（背景/局部/反应镜/纵深虚焦/前景遮挡/环境音留痕均可）
- 出场人物用对应的资产名称代替

### 4. 群演处理

- 群演不抢主角戏，不给单独配台词
- 用前景具象人物的微动作（遮、瞥、垂、攥）服务当前戏核情绪
- 后方群演身影隐入背景

### 5. 台词零删改铁律

- 所有引号内台词、VO、系统播报、面板文字 100% 逐字搬运
- 禁止合并、禁止精简、禁止省略修饰词
- 分镜师只设计画面，不做台词二次创作

### 6. 台词时间分配

- 根据情绪和语气分配，不平均切分
- 台词按照 **4字/秒** 进行计算
- 停顿/反应时间加长 15% 缓冲

### 7. 画面描述规范

- 只描述：谁做了什么动作/姿态/表情/状态变化（汗湿、泪痕、衣衫凌乱、青筋暴起）
- 不写：服装、发型、长相（这些交给图片资产）
- 不含：景别/运镜等技术标注（这些在对应列中）

### 8. 声音只写两类

- ✅ 环境音 + 音效
- ❌ BGM、配乐、音乐——剧本里的【BGM】只读不抄

### 9. VO音画同步

- VO（旁白/内心独白/系统播报/面板文字/短信/弹幕/标语等一切文字信息）一律按普通台词处理
- 画面照常描写人物动作/反应/环境
- 文字内容 100% 原样写在分镜末尾的 VO 里，音画同步，不得遗漏

## 片段间过渡与连贯性设计

核心目标：消灭片段切换时的「跳跃感」。

### 动作的桥梁

- 相邻片段描述同一组人物连续动作时：前一片段结尾 = 动作起始态，后一片段首镜 = 动作进行时或完成时
- ❌ 错误：片段A「他握紧剑柄。」→ 片段B「他拔剑冲上前。」
- ✅ 正确：片段A「他的手猛地握住剑柄，指节泛白。」→ 片段B「利剑'锵'一声出鞘。」

### 情绪的接力

- 对话/冲突场景：前一片段结尾用反应镜/眼神/微表情为后一片段铺底
- 后一片段首镜承接铺垫进行强化或反转
- ❌ 错误：片段A「你走吧。」→ 片段B「他转身离开。」
- ✅ 正确：片段A「特写她说完后紧抿的嘴角和瞬间泛红的眼眶。」→ 片段B「他看着她强忍泪水的脸→挫败地垂下眼，转身。」

### 空间与视线的链接

- 对话结束后切换场景：利用空镜/视线引导/声音元素建立空间联系
- ❌ 错误：室内争吵结束 → 直接切第二天喧闹街市
- ✅ 正确：人物看向窗外 → 接窗外暴雨空镜（雨声延续0.5秒） → 叠化至次日街市大全景

### 台词与动作的黏合（音画跨段同步）

- 前段末尾声音延续到后段首镜，由下一画面承接
- ✅ 正确：片段A结尾「咚的一声沉闷撞击。」→ 片段B开头「特写地上青花瓷碗，还在微微打转。」

```markdown

### 第1组（约Xs）
| # | 画面描述 | 场景 | 景别 | 运镜 | 角色动作 | 情绪 | 光影氛围 | 音效 | 台词 |
|---|---------|------|------|------|---------|------|---------|------|------|
| 1 | ... | ... | ... | ... | ... | ... | ... | ... | ... |
| 2 | ... | ... | ... | ... | ... | ... | ... | ... | ... |

### 第2组（约Xs）
...
```

## 转入 prompt-gen 时

分镜表的每一行 → 麻薯提示词的一个切片段（`|` 切片段N）。
分镜表的「画面描述 + 场景 + 情绪 + 光影 + 景别」→ 麻薯的「画面描述」和「场景设计」。
分镜表的「台词 + VO」→ 麻薯的「配音指令」。
分镜表的「音效」→ 麻薯总述头部的音效声明。
