# 竞品短剧逆向拆解方法论

> 用途：当用户提供热门短剧文案（音频转文字或完整文案）要求分析时，系统化提取爆款公式。
> 流程：内容结构化 → 段落功能标注 → 节奏公式提取 → 爽点机制归类 → 对标方案生成

---

## 前置条件

| 需要什么 | 从哪里来 |
|---------|---------|
| 文案正文 | 用户提供（音频转文字/完整txt/附件） |
| 榜单数据 | `trend-scan` 实时红果榜单 |
| 平台规则 | `references/platform-compliance-2026.md` |

## 五步拆解流程

### 第一步：内容结构化

将原始文案按自然场景切分为 SS（Scene Segment），每个 SS 包含：

| 字段 | 说明 |
|------|------|
| 段号 | SS1, SS2, SS3 ... |
| 内容摘要 | 50字以内描述本段发生了什么 |
| 功能标签 | 开局/压抑/行动/展示能力/对比/打脸/收获/温暖/钩子 |
| 用户情绪 | 😣 😤 😊 🤩 🥰 等 |

### 第二步：段落功能标注

对每个 SS 标注在整季中的叙事功能：

```
SS1  苦难开局 → 建立共情
SS2  主角内心独白 → 亮出金手指
SS3  主动选择 → 第一次展示主角性格
SS4  新环境嫌弃 → 制造期待反转
SS5  局部温暖破冰 → 给用户喘息
```

### 第三步：节奏公式提取

统计全季各功能段落比例，得出节奏公式。用标准格式输出：

```
[压抑%] + [行动%] + [困难/质疑%] + [解决/打脸%] + [钩子%]
```

**示例（发配边关 S1）：**
```
10% 压抑 + 20% 主动行动 + 40% 质疑/对比 + 20% 解决/收获 + 10% 钩子
```

### 第四步：爽点机制归类

列出所有爽点类型及其出现频次：

| # | 爽点类型 | 频次 | 说明 |
|---|---------|------|------|
| 1 | 主动选择 | 贯穿全剧 | 主角永远在决策 |
| 2 | 对比打脸 | N次 | 对照组1v1 |
| 3 | 能力展示→即时收获 | N次 | 每次展示能力紧跟物质回报 |
| 4 | 家庭认可 | N次 | 婆媳/弟妹关系线 |

### 第五步：对标方案生成

**⚠️ 对标不创新铁律（用户多次强调）：**

拆解的目的是 1:1 精确映射，不是创意发挥。当用户提供已验证的爆款作品作为对标参考时，只修改故事设定（时代/背景/金手指类型），**绝不修改以下要素**：

| 不可修改 | 可修改 |
|---------|--------|
| 情绪曲线 | 时代背景（古代→冰原/修仙/星际） |
| 节奏公式（压抑/打脸比例） | 金手指类型（木系异能→农业知识/兽医） |
| 爽点排列顺序 | 角色名字/社会身份 |
| 角色关系结构（对照组设计） | 世界观细节 |
| "不憋屈"原则（负面≤15%） | 具体剧情事件 |

**核心原则：如果原版是上行波动情绪线，对标版也必须是上行波动；不能改成深V反转。如果用户说"能不能也结合一下X"，在融合时必须保持原版的情绪曲线和节奏公式不变，只在外围添加X的元素。**

## 产出文件结构

拆解完成后，保存到 Obsidian 项目目录：

```
📁 <项目名>爆款拆解/
├── 00-爆款基因完整拆解（总纲）.md   ⭐ 20因子 + 1:1映射模板
├── 01-<名称>-逆向拆解报告.md        S1逐段拆解
├── 04-S1至S3节奏演进拆解.md         节奏变化分析
├── 06-S4至S6演进分析.md             后续赛季新增公式
├── 03-文案分析原始笔记.md           角色/话术/金句
└── README.md                        索引
```

## 六季演进预期

| 季 | 冲突对象 | 经营阶段 | 冲突性质 |
|:--:|:--------|:--------|:---------|
| S1 | 对照组（1v1） | 零→初收获 | 个人对比 |
| S2 | 群像（1vN） | 起步经营 | 群像嫉妒 |
| S3 | 全村质疑 | 规模化+分销 | 集体不信任 |
| S4 | 跟随者 | 产业升级 | 商业竞争 |
| S5 | 社会压力 | 多线经营 | 社会关系 |
| S6+ | 外部势力 | 规模化扩展 | 暴力冲突 |

每季解锁一个新维度，不是重复打脸。
