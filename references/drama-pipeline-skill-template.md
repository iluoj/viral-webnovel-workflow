# 短剧全链路转换模板

本 skill 定义小说→短剧→分镜提示词的全链路转换命令，供本profile内部直接使用。

## 管道命令

|- `drama-adapt`: Load `references/drama-adaptation.md` plus `references/platform-compliance-2026.md`. Convert webnovel chapters into △-format short-drama episodes with paywall placement and compliance filtering.

| `prompt-gen`: **⚠️ 前置步骤：必须先执行 style-select，向用户提供至少3个视觉风格选项，确认后再进入 prompt-gen。不得默认选择一个风格直接生成。**

  Load `references/matsuri-director-prompt.md` (麻薯动漫导演分镜格式 — 当前标准格式，mantoufan/seedance时间轴格式已退役).

  输入：△格式剧本 epXXX.md + 全局资产表 (02-角色资产表.md, 03-场景资产表.md).
  **两层独立输出：**

  **层1：素材清单** → `98-视频分镜提示词/epXXX-素材清单.md`
    格式：C##/S##/P## 资产编号 + **结构化分字段描述**（详见 `references/asset-description-format.md`，2026-06起强制使用新结构化格式，旧一段式格式已退役）

    #### 角色描述核心规则：
    - 字段：年龄/性别/年代/身份/外貌/发型/服装（上身/下身/鞋子）
    - ❌ **禁止情绪词** — 不写\"眼神锐利\"\"表情坚定\"\"冷静\"\"机灵\"等。外貌只写形状、颜色、状态、质地
    - 多套造型 = 多个独立角色条目（如「陆晚（早期·流放）」和「陆晚（中期·工装）」是两个不同C编号）
    - 群演也按完整结构写；旁白不属于角色

    **关键规则：资产裁剪** — 从全局资产表中选出本集使用的子集，同时明确标注本集**未出场**的资产（防止子任务编造）。示例：

  **层2：单段分镜提示词** → `98-视频分镜提示词/epXXX-段NNN-提示词.md`
    ⚠️ 核心规则：每15秒一段，**每段一个独立文件**，绝不合并。
    **文件名必须带「段」字**：`epXXX-段NNN-提示词.md`

    每个文件严格遵循麻薯动漫导演分镜格式（matsuri-director-prompt.md，而非seedance时间轴格式）：

    **总述头部格式：**
    ```
    有音效（[至少3种音效，顿号分隔]），无音乐，无字幕，运镜丝滑，人物动作流畅，有中文对话

    场景：[内/外]景 [具体地点] + [时间] + [天气/氛围]

    景别+拍摄角度：[景别列表]，[角度列表]。

    画面描述：[1-3句话概括核心画面，包含人物动作和情绪]

    场景设计：[导演视角分析：视觉焦点、人物与环境关系、光影对比、矛盾冲突视觉化、后续铺垫]

    目的标签：[情绪加压/矛盾爆发/行动交代/氛围铺垫/情绪释放/信息交代/转折铺垫/高潮反转]

    片段数：X个

    片段说明：1. [核心动作] 2. [核心动作] 3. [核心动作] ...
    ```

    **分段描述格式（每段4-5个|切片段）：**
    ```
    |切片段N-[详细画面描述：人物动作、表情、肢体语言、环境细节、氛围渲染]

    运镜补充：【[景别]/[角度]】，从[起幅画面]到[落幅画面]，速度[速度节奏描述]，跟焦[焦点对象和变化]，落点在[最终定格画面]。

    【角色名（对白）】
    （配音指令：[性别年龄]，[音色类型]｜[语气情绪]，"[台词]"）
    ```

    **铁律清单：**
    - 🔴 **运镜选择必须查表，不可凭直觉随机** — 按叙事目的查 `references/camera-shot-selection.md` 决策表选取合适的景别/角度/运动（选择顺序：目的标签 → 景别 → 角度 → 镜头运动）。输出只用用户能理解的简单术语：推/拉/摇/移/固定/正反打/仰拍/俯拍。禁止使用高级术语（如希区柯克变焦）
    - 🔴 **运镜六要素不可缺省**：景别/角度、起幅、落幅、速度、跟焦、落点
    - 🔴 **配音指令四要素**：人物特征、音色类型、语气情绪、台词含引号
    - 🔴 **OS/独白处理**：用【角色名（OS）】标记，不需要画面角色对口型
    - 🔴 **无字幕规则**：总述头部第一行必须含「无音乐，无字幕」
    - 🔴 **有中文对话**：必须标注，即使该段实际无对白也要保留此6个字
    - 🔴 **场景一致**：同一段内场景不切换，段落间才切
    - 🔴 **每段15秒+4-5个|切片段**：15秒内4-5个镜头切换，每个约3-4秒。禁止合并多段到一个文件
    - 🔴 **禁止分析**：仅输出结论，不解释不说明不思考过程

  **批量处理技巧（多集并行）：**
  当需要为多集（如 ep002-ep005）同时生成提示词时，使用 `delegate_task` 的 tasks 数组模式并行处理：
  ```
  delegate_task(tasks=[
    {goal: "为 ep002 生成提示词", context: "模板规则+剧本路径"},
    {goal: "为 ep003 生成提示词", context: "..."},
    {goal: "为 ep004 生成提示词", context: "..."},
  ])
  ```
  每集子任务产出11个文件（1素材清单 + 10段提示词）。然后处理剩余集。
  最高并行度：用户配置的 delegation.max_concurrent_children（默认3）。

| `compliance-audit`: Load `references/platform-compliance-2026.md` + `references/compliance-audit-template.md`. Run the full audit with 6-section output format: (1) 绝对禁止题材检查 (2) 高风险题材检查 (3) 六大零容忍红线检查 (4) 隐性红线检查 (5) 差异化审核尺度检查 (6) 风险项与整改建议. Output: `99-合规自检报告.md`. Can reject and route back to `draft-5` or `outline-30`.

### 并行批处理（多集同时出提示词）

当需要为多集（如 ep002-ep005）批量生成提示词时，使用 `delegate_task` 的 tasks 数组模式并行加速：

```
delegate_task(tasks=[
  {goal: "为 ep002 生成提示词", context: "模板规则+剧本路径+资产表路径…"},
  {goal: "为 ep003 生成提示词", context: "同上，换ep003"}…,
  {goal: "为 ep004 生成提示词", context: "同上，换ep004"}…,
])
```

每个子任务独立运行，产出 11 个文件（1 素材清单 + 10 段提示词），写入共享工作目录。剩余集（如 ep005）随后单独处理。最高并行度 = `delegation.max_concurrent_children`（默认 3）。

**注意事项：**
- 每个子任务是独立 agent，需要完整的上下文（模板规则、文件路径、风格描述、资产裁剪规则）
- 子任务产出文件到共享工作目录，父 agent 不需要合并
- 必须显式告知子任务：使用麻薯动漫导演分镜格式（总述头部+|切片段），而非 seedance 时间轴格式
- 必须显式告知子任务：资产裁剪规则（标注未出场资产）、运镜六要素、配音指令四要素、OS/独白处理规则
- ⚠️ 子任务容易少写切片段（写3个而非4-5个）— context 中必须双重强调每段4-5个|切片段
- ⚠️ 子任务容易漏「有中文对话」标记 — 即使无对白也必须保留
- ⚠️ 素材清单必须使用新结构化格式（asset-description-format.md），禁止复制全局资产表中的旧格式
- ⚠️ 子任务context中优选 _prompt-gen-rules.md 文件引用而非内嵌完整规则（节省token+减少错误）
- ⚠️ Windows下批量生成后必须清理残留进程：taskkill //F //IM python.exe；taskkill //F //IM bash.exe
- 每个子任务通过 read_file 读取剧本+资产表，通过 write_file 写入输出

⚠️ 当前分镜格式标准见 `references/matsuri-director-prompt.md`（麻薯动漫导演分镜格式）。mantoufan/seedance（时间轴格式）已退役。

## 工作区约定

- 源小说文件位于用户指定的 Obsidian 工作区（如 `E:\ObsidianSource\03_AI漫剧\<小说名>\02_正文_第X章.md`）
- 输出目录固定为 `<小说名>\短剧版\`
- 输出结构：
  ```
  短剧版/
  ├── 01-题材定位.md          # S/A/B/C分级 + 受众 + 付费策略
  ├── 02-角色资产表.md        # 角色编号C01-C99 + 外貌/性格标签
  ├── 03-场景资产表.md        # 场景编号S01-S99 + 风格关键词
  ├── 04-全集大纲.md          # 60-100集 × 三幕结构表
  ├── 05-付费卡点布局.md      # 付费卡点（本流程已移除，所有集免费）
  ├── 99-合规自检报告.md      # 红线 + 灰区全维度扫描（6章节格式：绝对禁止题材+高风险题材+零容忍红线+隐性红线+差异化审核尺度+风险项）
  ├── episodes/               # △格式文字剧本（drama-adapt产出）
  │   ├── ep001.md
  │   └── ...
  └── 98-视频分镜提示词/       # 分镜提示词（prompt-gen产出）
      ├── ep001-素材清单.md
      ├── ep001-段001-提示词.md
      └── ...
  ```

## 输入来源

从用户指定的 Obsidian 工作区读取小说文件，输出剧本/提示词到短剧版目录。

```
写小说 → *.md → drama-adapt（转△剧本）→ prompt-gen（分镜提示词）
        ↘ compliance-audit（审核把关）
```
