# 短剧剧本转换指南（△格式）

> 🎯 用途：ag_014 drama-adapt agent 的核心转换参考
> 📚 方法论来源：0xsline/short-drama (MIT) 短剧剧本创作框架
> ⚠️ 能力边界：本指南仅覆盖△格式剧本转换。不涉及视频/AI分镜提示词生成。

---

## 一、转换总则

### 核心差异

| 维度 | 网文 | 短剧 |
|------|------|------|
| 篇幅 | 2000-3000字/章 | 1-3分钟/集 |
| 叙事 | 叙述+对话+内心独白 | △镜头描述+对白+OS/VO |
| 节奏 | 可铺垫可展开 | 前30秒必须钩住观众 |
| 世界观 | 可详细解释 | 用画面展示，不解释 |
| 付费 | 按章节付费 | 全集免费放送 ← 本流程移除付费卡点 |
| 类型 | 番茄/七猫风格 | 即梦/小云雀/抖音漫剧 |

### 单集时长规范

**每集时长：2分-2分30秒，硬上限3分钟。** 具体规则：
- 目标区间：**120秒-150秒**
- 硬上限：**180秒**（超过必须压缩）
- 控制方法：每集约35-40个|切片段，每个切片段约3-4秒
- 在△剧本头部标注的`**时长**`必须与实际内容匹配（按字秒换算或分镜计数）

### 格式转换规则

```
网文章节 → 短剧单集
  叙述性文字 → △ 镜头描述（全景/中景/特写）
  对话 → 角色名（语气）：对白
  内心独白 → OS（画外音）或 VO
  时间跳跃 → 黑屏字幕 "三日后"
  世界观的确认 → 场景切换前标注时间地点
  章末钩子 → 本集完
```

### △ 镜头描述标记

```
△（全景）     —— 交代空间和环境
△（中景）     —— 角色互动，对话场景
△（近景）     —— 强调表情和情绪
△（特写）     —— 关键物品、表情细节、信息揭示
△（闪回）     —— 回忆段落
△（分屏）     —— 同时展示两个场景
```

### 对白标记

```
角色名（语气）："对白内容"
  — 语气选项：低声/颤抖/冷笑/怒吼/哽咽/平静/兴奋/迟疑
  — OS = 画外音（角色独白，口型不同步）
  — VO = 旁白（叙事者，非角色）
```

### 单集结构模板

```markdown
# ep{编号} - {集标题}

**时长**：{X分X秒}

---
△（场景/镜头描述）

**角色名**（语气）："对白"

△（转场/新镜头描述）

【本集完】
```

## 改编8大核心要点（蒸馏自 Toonflow）

网文→短剧改编的一切决策须以此8条为基准：

1. **强画面感（可拍摄性）**：确保所有保留内容能转化为镜头语言，拍不出来则换表达方式。
2. **台词精简（高信息密度）**：剔除冗余，每句台词须服务于剧情推进或人物塑造；用台词传递背景信息。
3. **节奏极致快**：每一个画面都拉升情绪，可适当牺牲细微逻辑，优先保证节奏紧凑。
4. **只沿主线展开**：摒弃多支线，所有情节围绕单条主线推进；砍掉支线，仅保留核心人设与高光时刻。
5. **降低理解成本**：世界观不复杂，观众听台词就能掌握核心剧情，漏看部分不影响整体理解。
6. **情绪大于一切**：无需复杂人物弧光，核心提供饱满强烈的情绪体验；逻辑与情绪冲突时优先保障情绪张力。
7. **开篇给足期待感**：第1集呈现激烈、高情绪张力场景，后续围绕开篇建立的期待感展开。
8. **展示不要告诉**：闭坑「自爆家门式台词」，能靠一个动作/眼神传递的信息坚决不用嘴说；把原著的叙述/心理描写转成可拍的动作与画面（动作是因、对话是果）。

### 删减决策优先级

**优先删除**：节奏拖沓的铺垫场景 / 信息密度低的重复内容 / 载体不支持的内容（大段心理描写） / 主线贡献弱的支线。
**优先保留**：核心情绪点（爆点/虐点/爽点）/ 关系拉扯场景 / 付费点前情绪铺垫 / 身份反差与信息差场景 / 高光打脸时刻。

## 二、四阶段节奏波形（60集为例）

### 各阶段集数分布

| 阶段 | 集数范围 | 免费/付费 | 节奏特征 |
|------|----------|-----------|----------|
| 起势段 | 1-9集 | ✅ 全部免费 | 快速建立人物+世界，每2-3集一小高潮 |
| 攀升段 | 10-27集 | ✅ 全部免费 | 主线冲突展开，爽点密集区 |
| 风暴段 | 28-45集 | ✅ 全部免费 | 中型爽点+感情线推进，每5-7集一中高潮 |
| 决战段 | 46-60集 | ✅ 全部免费 | 终极对决，结局大团圆 |

### 每集微结构

```
前30秒（钩子段）：
  ├── 承接上集悬念（非第1集）
  ├── 抛出本集核心冲突
  └── 建立情绪基调

中间1-1.5分（冲突升级段）：
  ├── 场次一：冲突展开
  ├── 场次二：压力升级
  └── 场次三：转折/新信息

最后30秒（爽点/钩子段）：
  ├── 方案A：爽点释放+新悬念（常规集）
  └── 方案B：极致悬念+硬切（高潮集）
```

## 三、压抑-释放周期压缩规则（网文→短剧跨集节奏）

> 网文可以铺垫2-3章再给爽点，短剧不行。转换时**必须主动压缩**。

### 压缩目标

| 维度 | 网文原文节奏 | 短剧应压缩为 |
|------|------------|------------|
| 连续压抑集数 | 可容忍2-3章 | **最多1-2集，第3集必须有释放** |
| 释放间隔 | 每3-4章1次 | **每2集至少1次** |
| 同类型冲突 | 可连续出现 | **相邻2集不能是同类型冲突** |

### 压缩方法

- 🔪 **删除冗余铺垫**：将3章的压抑弧线合并为1-2集。跳过"主角犹豫/反复确认/等待时机"等中间状态
- ⏩ **前置释放**：如果第1集制造了冲突，第2集就给观众一点甜头或进展（哪怕只是一个小胜利或秘密揭露）
- 🔄 **合并同类项**：连续3集都是"反派打压→主角忍"的场景 → 砍掉中间那集，或把第三集的"忍"改为"第一次反击"
- ✅ **释放不必是大爽点**：一个秘密的揭开、一个意外的援军、一句关键对话都算释放，不一定要大打出手
- 🚫 **3集定律**：任何连续3集压抑无释放的弧线，必须砍掉至少1集或在第3集插入释放

### 与四阶段波形配合

不是推翻四阶段波形，而是在每阶段内部按此规则压缩：

```
起势段（1-9集）：快速建立，每2集1个小高潮  ← 加速记忆
攀升段（10-27集）：爽点密集，每2集至少1个释放
风暴段（28-45集）：中型爽点，最多1-2集压抑
决战段（46-60集）：节奏拉满，不允许连续2集无进展
```

## 四、5大爽点类型到短剧适配

| 网文爽点 | 短剧爽点 | 情绪公式 |
|----------|----------|----------|
| 打脸装逼 | 身份碾压+打脸复仇 | 轻视→蓄力→揭露→碾压→跪地 |
| 突破升级 | 逆袭翻盘 | 谷底→绝望→转机→翻盘→巅峰 |
| 宝物奇遇 | 逆袭翻盘（转机出现） | 困境→意外获得→破局 |
| 感情升温 | 甜蜜撒糖（甜宠专属） | 试探→靠近→心动→确认 |
| 反派被制裁 | 打脸复仇+真相揭露 | 嚣张→蓄力→反杀→当众出丑 |

## 五、每集时长标准

### 时长要求（用户指定，硬性规则）

| 维度 | 要求 |
|:----|:-----|
| ✅ **目标区间** | **2:00 ~ 2:30**（120s ~ 150s） |
| ⚠️ **硬性上限** | **3:00**（180s），超过必须压缩 |
| ⬆️ **不足处理** | 低于2:00的集必须扩充内容 |

### 时长估算方法

用脚本内容量估算时长，公式如下：

```
描述字 ÷ 4.5   = 场景描述阅读时间
+ 对白字 ÷ 3.0 = 对白说话时间（含停顿）
+ OS条数 × 4   = 独白时间
+ △条数 × 0.5 = 镜头切换间隙
+ 场景转场数 × 1.5 = 转场时间
+ 4.0           = 片头片尾留白

≈ 总时长（秒）
```

**粗略经验值：** 每集脚本内容约600-900中文字（含△描述、对白、OS）可达到2:00-2:30。少于500字的集基本低于1:30，需要扩充。

### 扩充方法

当单集内容量不足2:00时，通过以下方式增加（不改剧情结构）：

| 方法 | 每处加时 | 说明 |
|:----|:--------:|:-----|
| △描述加视觉细节（环境/光影/表情） | +5~8秒 | 场景描述增加2-3句细节描写，如"晨光斜照在灶台上，油锅里还在滋滋响" |
| 增加对话来回 | +8~12秒/次 | 在一段对话中加1-2句一来一回的应答 |
| 增加OS内心独白 | +3~5秒/条 | 在关键节点插入角色的OS心理活动 |
| 延长镜头停留（画面留白） | +2~3秒/段 | 在情绪高潮或转折处加入无台词的镜头停顿 |

### 时长标注规范

每集△格式脚本必须标注时长，格式统一为：

```markdown
**时长**：2分30秒
```

用时估算方法计算后填入。允许±15秒的偏差。不建议统一标注一个默认值（如全部标"2分30秒"），应逐集估算后填写。
