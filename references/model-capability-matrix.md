# 模型能力矩阵 — 创作流水线选型依据

> 创建日期：2026-06-27
> 用途：为 viral-webnovel 6-profile 流水线提供模型选型的量化参考
> 前提：所有主流模型（Gemini 3.1 Pro、deepseek-v4-pro、deepseek-v4-flash）都支持 1M context。差异在推理质量，不在 context 大小。

## 核心维度定义

| 维度 | 定义 | 在创作流水线中的含义 |
|------|------|-------------------|
| 长上下文推理 | 在 1M context 中保持段间连贯性，不丢失叙事线索 | 分镜提示词需要理解全集情绪弧线 |
| 视觉想象力 | 文字 → 画面的转换能力 | 分镜描述、场景光影、角色神态 |
| 格式严格遵循 | 按指定模板输出的精确度 | △剧本格式、麻薯分镜格式 |
| 中文创意表达 | 中文散文质感、对白自然度、情节创意 | 小说正文、剧本对白 |

## 任务 → 模型映射

| 任务 | 核心需求 | 首选 | 理由 |
|------|---------|------|------|
| 分镜提示词 (prompt-gen) | 长上下文推理 + 视觉想象 + 格式 | Gemini 3.1 Pro | 三项全部顶格 |
| △剧本 (drama-adapt) | 中文创意 + 格式 | Gemini 3.1 Pro | 对白自然度、情感层次 |
| 圣经/大纲 (bible/outline) | 中文创意 | Gemini 3.1 Pro | 发散不套路 |
| 小说正文 (draft-5) | 中文散文质感 | deepseek-v4-pro 或 GPT-5.5 | 稳+中文底子好 |
| 趋势/选题 (trend/idea) | 推理分析 | deepseek-v4-pro | 中文推理最优 |
| 合规/诊断 (audit/review) | 规则匹配 | deepseek-v4-flash | 不需要创意，性价比最高 |

## ⚠️ 已修正的错误认知

| 错误 | 修正 |
|------|------|
| "Gemini 1M context 是独家优势" | deepseek v4 系列也是 1M。差异在推理质量 |
| "分镜提示词是模板填充，用 flash 就行" | 分镜是导演模式——需要全集上下文推理，flash 不够 |
| "prompter 属于执行层" | prompter 属于创意层——需要视觉想象力+长上下文推理 |
